# Lyrics Next 格式规范第二版

> [!note]
> 仅供娱乐，当然如果你的软件想使用这个格式是完全可以的（真的会有人用吗？）

一种基于 [LRC](https://zh.wikipedia.org/wiki/LRC%E6%A0%BC%E5%BC%8F) 的逐音节歌词格式，参考了 [TTML](https://en.wikipedia.org/wiki/Timed_Text_Markup_Language)、[Lyricify Syllable](https://github.com/WXRIW/Lyricify-App/blob/main/docs/Lyricify%204/Lyrics.md#lyricify-syllable-%E6%A0%BC%E5%BC%8F%E8%A7%84%E8%8C%83) 和 [Apple Syllable](https://help.apple.com/itc/videoaudioassetguide/#/itc0f14fecdd)  
Lyrics Next 为了解决普通 TTML 歌词可读性差、难以直接编辑的问题而诞生  
Lyrics Next 歌词文件可以直接使用 `lrc` 后缀，也可以使用单独的 `lrcn`

#### *带 * 的条目为非强制要求*

## 更改

相较于原本的 Lyrics Next 第一版，*Lyrics Next 第二版* 进行了许多更改。  
我们认为读者已经了解第一版的特性，故下列变更会基于第一版的改动进行描述。

- 对头部信息进行了精简和变更，使其更适配 Apple Music 新特性
- 将 `end` 标签统一为同步多媒体集成语言（SMIL）表示
- 大幅修改了特殊歌词的格式，使其更适配 Apple Music 新特性

## 规范用语

### 时间标签

在 Lyrics Next 中，歌词行行首被 `[]` 包裹的内容被称为 `行标签`，后续的歌词内容被称为 `正文`  
正文中被 `<>` 包裹的内容被称为 `逐音节标签`  
凡事使用时间轴标记的内容均可称为 `对象`，包括行，字，词或音节  

### 歌曲对唱关系

根据音乐中常见的演唱方式，歌词分为 `主唱(人声/歌词)` 与 `背景(人声/歌词)`  

- 主唱: 常规的 `agent` 均为 `主唱人声`，属于歌词正文，用于主要歌词  
- 背景: `agent` 带有 `x-bg` 字样为 `背景人声`，一般用于背景声。

背景人声的歌词位于主唱歌词下方，字体较小，该歌词行称为 `背景人声` `背景行` `bg 行`

## 头部信息 *

歌词头部可选的包含歌词相关信息，如：

| 标签                           | 含义                                                     |
| ------------------------------ | -------------------------------------------------------- |
| [lrcn-vertion:v2.0]            | Lyrics Next 格式版本                                     |
| [timing:syllable]              | 项目类型<br>`line` 为逐行，`word` `syllable` 为逐字/音节 |
| [lang:zh-CN]                   | 主要语言                                                 |
| [translates.lang:en-US]        | 翻译语言                                                 |
| [transliterations.lang:pinyin] | 发音语言                                                 |
| [songwriter:HOYO-MiX]          | 制作人名称                                               |
| [agent.v1:person]              | `agent` 对应信息                                         |
| [agent.v2:group]               | 同上                                                     |
| [by:MiaowCham]                 | 歌词制作者                                               |
| [offset:0]                     | 歌曲偏移量（单位毫秒）                                   |

若一个标签有多个内容，可以用 `,` 分割或者使用多个标签表示

>[!note]
该内容可以不填，不要包含重复标签。(重复标签指完全相同的标签)  
若希望得到完整的类 Apple Syllable 体验，需填写 `songwriter` `agent` 对应信息

## 歌词

Lyrics Next 歌词的标准格式为：

```lrcn
[start,end,agent]<start,end>Word <start,end>word
```

- `start` 为起始时间，其时间戳使用同步多媒体集成语言（SMIL）表示。  
    一般情况下使用SMIL表示，可提供完整或部分时钟值：  
    `hh:mm:ss.sss（时:分:秒.毫秒）`  
    *其中，小时和毫秒可不提供。*  
- `end` 为时长，其时间戳使用同步多媒体集成语言（SMIL）表示。  
    *该对象可以省略，若不存在则将后一个项目的开始时间作为结束时间。*
- `agent` 用于标记该句演唱歌词的艺人的 ID。  
    每一位演唱者都应该被分派到一个ID。
    *该对象可以省略，若只有一个演唱者可以不填。*
- `<>` 中的时间戳为后方单词的起始时间和时长。

### 省略机制

为兼容 LRC 与 增强型LRC 格式，*Lyrics Next 第二版* 要求允许对额外扩展的对象进行省略。  
因此，Lyrics Next 格式是 LRC 与 增强型LRC 格式的超集，所有标准的 LRC 与 增强型LRC 格式均为合法的 Lyrics Next 格式歌词

#### 行标签中，*Lyrics Next 第二版* 额外扩展了 `end` 和 `agent` 标签，允许进行以下类型的省略：

- `[start,end]`（只省略 `agent` 标签，单一演唱者时可以使用）
- `[start,,agent]`（只省略 `end` 标签）
- `[start]`（全部省略，LRC 格式）

为了保持格式完整，只省略 `end` 标签时，必须保留逗号。  

省略 `end` 标签时，解析应当遵循：

- 有逐音节的，将本行最后一个音节的结束时间作为结束时间
- 逐行的，以下一行的开始时间作为结束时间

#### 逐音节标签中，*Lyrics Next 第二版* 额外扩展了 `end` 标签，允许省略：

- `<start>`（全部省略，增强型LRC 格式）

### 逐句歌词  

如果想使用 Lyrics Next 记录逐句而非逐音节歌词，则无需逐词标记时间信息：

```lrcn
[start,end,agent]Lyrics
```

> 同样的，你可以根据 [省略机制](#省略机制) 对行标签的对象进行省略，但这样的话和普通 LRC 区别在哪？

## 附属歌词

包括背景人声、翻译/发音信息，这些内容均视为主唱歌词的附属歌词  
`x-bg` 对应背景人声，`translate` 对应翻译行，`transliteration` 对应发音行

### 背景人声

背景人声必须在主唱歌词下方，具体格式如下：

```lrcn
[start,end,agent]<start,end>Word <start,end>word  # 主唱歌词
[x-bg]<start,end>Word <start,end>word               # 背景
```

> [!note]
> 即使背景人声的起始时间在主句前面，背景人声也得在主句的后一行

### 翻译及行发音

内嵌：翻译及行发音应当位于其所属行下方，具体格式如下：

```lrcn
[start,end,agent]<start,end>Word <start,end>word  # 主唱歌词
[translate]翻译
[transliteration]音译
[bg]<start,end>Word <start,end>word               # 背景
[translate]背景翻译
[transliteration]背景音译
```

外置：为了满足 i18n 需求，建议尽可能使用外置翻译/发音。  
外置翻译/发音文件应当是 LRC 格式文件，具体格式如下：

```lrc
[type:translate]   # 声明类型
[lang:zh-CN]       # 声明语言
[start]内容
...
```

其中，`start` 标签必须和原文的开始时间一致。若存在多句共用完全相同的开始时间，则按照上下顺序对应。

> [!note]
> 外置方案是将翻译/发音单独存放文件，通过文件名或专辑元数据文件进行关联。  
> 关于关联方式，不属于本格式规范范畴，可以参考 [Apple Music 技术规范](https://help.apple.com/itc/musicspec/#/itccdd938379)。  

### 逐音节发音 *

*Lyrics Next 第二版* 支持记录 iOS 26 起的逐音节发音。  
逐音节发音格式类似背景人声，但发音的音节必须和其所属的音节对应，具体格式如下：

```lrcn
[transliteration]<start,end>Word <start,end>word
```

外置发音文件时，`[transliteration]` 需要变更为和所属行对应的 `[start]`。
